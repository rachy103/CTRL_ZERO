# 튜닝 가이드

튜닝은 `main.py` 상단 `USER TUNING PARAMETERS` 영역에서 합니다. 기능 구현 모듈에는 기본 동작만 있고, 실차 값은 `main.py`에서 주입합니다.

## 권장 순서

1. `vision + opencv`로 카메라 방향과 화면 ROI를 맞춥니다.
2. `manual + opencv`로 Arduino 모터 방향을 확인합니다.
3. `vision + yolo`로 YOLO 차선 overlay를 확인합니다.
4. `auto + yolo`를 낮은 속도에서 시작합니다.
5. LiDAR를 켜고 정지 거리와 감속 거리를 조정합니다.

## 조향 튜닝

현재 자동 조향은 Stanley feedback에 곡률 feed-forward를 더한 방식입니다.

```python
K_STANLEY = 1.6
HEADING_GAIN = 1.0
CURVATURE_FF_GAIN = 1.0
STEER_SCALE = 100.0
STEER_SLEW_BASE = 18.0
STEER_SLEW_MIN = 8.0
```

- 차선 중앙에서 벗어났을 때 복귀가 약하면 `K_STANLEY`를 올립니다.
- 커브 진입이 늦으면 `CURVATURE_FF_GAIN`을 조금 올립니다.
- 조향이 너무 급하면 `STEER_SCALE`을 낮추거나 `STEER_SLEW_BASE`, `STEER_SLEW_MIN`을 낮춥니다.
- 조향 방향이 반대면 `REVERSE_STEER = True`로 바꿉니다.

## 속도 튜닝

```python
MIN_SPEED = 35
MAX_SPEED = 80
CURVE_SPEED_GAIN = 3.0
KAPPA_REF = 0.0015
```

- 처음에는 `MAX_SPEED`를 낮게 둡니다.
- 직선에서 안정적이면 `MAX_SPEED`를 5씩 올립니다.
- 커브에서 밀리면 `CURVE_SPEED_GAIN`을 올려 곡률 기반 감속을 키웁니다.
- 화면에 표시되는 `kappa`가 급커브에서 어느 정도인지 보고 `KAPPA_REF`를 맞춥니다.

실행 중 `+`, `-` 키로 직선 목표 속도를 임시 조정할 수 있습니다.

## 차선 신뢰도 기준

```python
MIN_LANE_CONFIDENCE_TO_DRIVE = 0.45
```

차선이 불안정한데 계속 출발하면 이 값을 올립니다. 차선이 보이는데도 너무 자주 멈추면 조금 낮춥니다.

## ROI crop / Bird-eye view 튜닝

YOLO 입력 전처리는 `main.py` 상단에서 조정합니다. ROI crop은 화면에서 도로가 아닌 영역을 잘라 FPS와 오검출을 줄이는 용도입니다.

```python
ROI_ENABLED = True
ROI_TOP_RATIO = 0.35
ROI_BOTTOM_RATIO = 1.00
ROI_LEFT_RATIO = 0.00
ROI_RIGHT_RATIO = 1.00
```

- `ROI_TOP_RATIO`는 위쪽을 얼마나 버릴지 정합니다. `0.35`면 화면 위 35%를 제외하고 아래 65%만 YOLO에 넣습니다.
- 차선이 너무 가까운 부분만 잡히면 `ROI_TOP_RATIO`를 낮춥니다.
- 하늘, 벽, 책상 같은 배경 오검출이 많으면 `ROI_TOP_RATIO`를 올립니다.
- 카메라가 한쪽으로 치우쳤으면 `ROI_LEFT_RATIO`, `ROI_RIGHT_RATIO`로 좌우를 조금 잘라냅니다.

Bird-eye view는 원근을 펴서 차선을 위에서 본 형태로 만드는 기능입니다. 카메라 높이와 각도에 따라 기준점이 달라지므로 처음에는 꺼둔 상태로 ROI부터 맞춘 뒤 켭니다.

```python
BIRD_EYE_ENABLED = False
BIRD_EYE_SRC_BOTTOM_LEFT = (0.10, 0.98)
BIRD_EYE_SRC_BOTTOM_RIGHT = (0.90, 0.98)
BIRD_EYE_SRC_TOP_RIGHT = (0.62, 0.35)
BIRD_EYE_SRC_TOP_LEFT = (0.38, 0.35)
BIRD_EYE_DST_MARGIN_RATIO = 0.18
```

- `BIRD_EYE_SRC_*`는 crop된 화면 기준의 사다리꼴 네 꼭짓점입니다. 값은 `(x_ratio, y_ratio)`이고 0.0~1.0 범위입니다.
- 화면에서 가까운 차선 폭은 `SRC_BOTTOM_*`, 먼 차선 폭은 `SRC_TOP_*`로 맞춥니다.
- Bird-eye를 켠 뒤 차선이 바깥으로 휘거나 중심이 튀면 기준점이 맞지 않는 것입니다. `--no-bird-eye`로 끄고 ROI부터 다시 맞춥니다.

## YOLO 차선 튜닝

```python
YOLO_IMAGE_SIZE = 640
YOLO_CONFIDENCE = 0.25
YOLO_IOU = 0.45
YOLO_FRAME_SKIP = 1
YOLO_MIN_POINTS_PER_LANE = 3
YOLO_MIN_VALID_Y_SPAN_RATIO = 0.08
YOLO_MASK_SAMPLE_STEP_PX = 6
YOLO_SKELETON_ENABLED = True
YOLO_SKELETON_BRIDGE_GAP_PX = 17
YOLO_DASHED_MERGE_MAX_X_GAP_RATIO = 0.12
YOLO_CURVE_FIT_DEGREE = 2
YOLO_SOLIDIFY_STEP_PX = 6
YOLO_FIT_OUTLIER_REJECTION_PX = 35.0
```

- FPS가 낮으면 `YOLO_IMAGE_SIZE`를 `416` 또는 `320`으로 낮춥니다.
- 여전히 느리면 `YOLO_FRAME_SKIP`을 `2`로 둡니다.
- 점선을 놓치면 `YOLO_CONFIDENCE`를 낮추고 `YOLO_MIN_VALID_Y_SPAN_RATIO`를 낮춥니다.
- 오검출이 많으면 `YOLO_CONFIDENCE`와 `YOLO_MIN_POINTS_PER_LANE`을 올립니다.
- 점선이 조각조각 잡히면 `YOLO_SKELETON_BRIDGE_GAP_PX`와 `YOLO_DASHED_MERGE_MAX_X_GAP_RATIO`를 조금 올립니다.
- 서로 다른 차선이 한 줄로 합쳐지면 `YOLO_DASHED_MERGE_MAX_X_GAP_RATIO`를 낮춥니다.
- 커브에서 차선 중심이 늦게 따라오면 `YOLO_CURVE_FIT_DEGREE = 2`를 유지합니다. 직선 트랙에서 흔들림이 크면 `1`로 낮춰도 됩니다.

## LiDAR 튜닝

```python
LIDAR_STOP_DISTANCE_MM = 450.0
LIDAR_SLOW_DISTANCE_MM = 900.0
LIDAR_MIN_SPEED_SCALE = 0.35
```

- `STOP_DISTANCE` 안에 장애물이 들어오면 속도는 0이 됩니다.
- `SLOW_DISTANCE` 안에서는 선형으로 감속합니다.
- 차가 너무 늦게 멈추면 두 거리 값을 모두 키웁니다.

## 로그 확인

로그를 켜려면 다음 값을 바꿉니다.

```python
LOG_ENABLED = True
```

또는 실행 중 `l` 키로 토글할 수 있습니다. 로그는 `lane_logs/drive_log.csv`와 `lane_logs/frames/`에 저장됩니다.
