# 튜닝 가이드

튜닝은 `main.py` 상단 `USER TUNING PARAMETERS` 영역에서 합니다. 기능 구현 모듈에는 기본 동작만 있고, 실차 값은 `main.py`에서 주입합니다.

## 권장 순서

1. `vision + opencv`로 카메라 방향과 화면 ROI를 맞춥니다.
2. `manual + opencv`로 Arduino 모터 방향을 확인합니다.
3. `vision + yolo`로 YOLO 차선 overlay를 확인합니다.
4. `auto + yolo`를 낮은 속도에서 시작합니다.
5. LiDAR를 켜고 정지 거리와 감속 거리를 조정합니다.

## 조향 튜닝

현재 기본 자동 조향은 contest 코드의 각도/위치 오차 가중합 방식입니다.

```python
CONTROL_MODE = "contest"
CONTEST_ANGLE_WEIGHT = 0.7
CONTEST_POSITION_WEIGHT = 0.05
MAX_STEER = 100
```

- 차선 중앙에서 벗어났을 때 복귀가 약하면 `CONTEST_POSITION_WEIGHT`를 올립니다.
- 커브 방향 반응이 약하면 `CONTEST_ANGLE_WEIGHT`를 올립니다.
- 조향이 너무 급하면 `CONTEST_POSITION_WEIGHT`나 `CONTEST_ANGLE_WEIGHT`를 낮춥니다.
- 조향 방향이 반대면 `REVERSE_STEER = True`로 바꿉니다.

## 속도 튜닝

```python
MIN_SPEED = 200
MAX_SPEED = 200
```

- 처음에는 `MAX_SPEED`를 낮게 둡니다.
- 직선에서 안정적이면 `MAX_SPEED`를 5씩 올립니다.
- contest 모드에서는 기본 주행 속도로 `MAX_SPEED`를 씁니다.
- `MIN_SPEED`는 실행 중 `-` 키로 낮출 때의 하한값입니다.

실행 중 `+`, `-` 키로 직선 목표 속도를 임시 조정할 수 있습니다.

## 차선 신뢰도 기준

```python
MIN_LANE_CONFIDENCE_TO_DRIVE = 0.45
```

차선이 불안정한데 계속 출발하면 이 값을 올립니다. 차선이 보이는데도 너무 자주 멈추면 조금 낮춥니다.

## ROI crop / Bird-eye view 튜닝

YOLO 입력 전처리는 `main.py` 상단에서 조정합니다. ROI crop은 화면에서 도로가 아닌 영역을 잘라 FPS와 오검출을 줄이는 용도입니다.

```python
ROI_ENABLED = True
ROI_TOP_RATIO = 0.35
ROI_BOTTOM_RATIO = 1.00
ROI_LEFT_RATIO = 0.00
ROI_RIGHT_RATIO = 1.00
```

- `ROI_TOP_RATIO`는 위쪽을 얼마나 버릴지 정합니다. `0.35`면 화면 위 35%를 제외하고 아래 65%만 YOLO에 넣습니다.
- 차선이 너무 가까운 부분만 잡히면 `ROI_TOP_RATIO`를 낮춥니다.
- 하늘, 벽, 책상 같은 배경 오검출이 많으면 `ROI_TOP_RATIO`를 올립니다.
- 카메라가 한쪽으로 치우쳤으면 `ROI_LEFT_RATIO`, `ROI_RIGHT_RATIO`로 좌우를 조금 잘라냅니다.

Bird-eye view는 원근을 펴서 차선을 위에서 본 형태로 만드는 기능입니다. 카메라 높이와 각도에 따라 기준점이 달라지므로 처음에는 꺼둔 상태로 ROI부터 맞춘 뒤 켭니다.

```python
BIRD_EYE_ENABLED = False
BIRD_EYE_SRC_BOTTOM_LEFT = (0.10, 0.98)
BIRD_EYE_SRC_BOTTOM_RIGHT = (0.90, 0.98)
BIRD_EYE_SRC_TOP_RIGHT = (0.62, 0.35)
BIRD_EYE_SRC_TOP_LEFT = (0.38, 0.35)
BIRD_EYE_DST_MARGIN_RATIO = 0.18
BIRD_EYE_MASK_SOURCE_POLYGON = True
YOLO_DISPLAY_BIRD_EYE_VIEW = True
```

- `BIRD_EYE_SRC_*`는 crop된 화면 기준의 사다리꼴 네 꼭짓점입니다. 값은 `(x_ratio, y_ratio)`이고 0.0~1.0 범위입니다.
- 화면에서 가까운 차선 폭은 `SRC_BOTTOM_*`, 먼 차선 폭은 `SRC_TOP_*`로 맞춥니다.
- `BIRD_EYE_MASK_SOURCE_POLYGON = True`이면 source 사다리꼴 밖은 검정으로 지운 뒤 YOLO에 넣습니다.
- `YOLO_DISPLAY_BIRD_EYE_VIEW = True`이면 `CTRL_ZERO` 화면과 mask 창도 BEV 기준으로 표시합니다.
- Bird-eye를 켠 뒤 차선이 바깥으로 휘거나 중심이 튀면 기준점이 맞지 않는 것입니다. `--no-bird-eye`로 끄고 ROI부터 다시 맞춥니다.

## YOLO 차선 튜닝

```python
YOLO_IMAGE_SIZE = 640
YOLO_CONFIDENCE = 0.25
YOLO_IOU = 0.45
YOLO_FRAME_SKIP = 1
YOLO_CLASS_NAMES = ("car", "lane1", "lane2", "traffic_light")
YOLO_SEGMENTATION_MODE = "lane_area"
YOLO_DRIVABLE_MIN_ROW_WIDTH_RATIO = 0.05
YOLO_DRIVABLE_EDGE_PERCENTILE = 2.0
YOLO_MIN_POINTS_PER_LANE = 3
YOLO_MIN_VALID_Y_SPAN_RATIO = 0.08
YOLO_MASK_SAMPLE_STEP_PX = 6
YOLO_SKELETON_ENABLED = True
YOLO_SKELETON_BRIDGE_GAP_PX = 17
YOLO_DASHED_MERGE_MAX_X_GAP_RATIO = 0.12
YOLO_CURVE_FIT_DEGREE = 2
YOLO_SOLIDIFY_STEP_PX = 6
YOLO_FIT_OUTLIER_REJECTION_PX = 35.0
```

- FPS가 낮으면 `YOLO_IMAGE_SIZE`를 `416` 또는 `320`으로 낮춥니다.
- 여전히 느리면 `YOLO_FRAME_SKIP`을 `2`로 둡니다.
- 현재 기본값은 `YOLO_SEGMENTATION_MODE = "lane_area"`입니다. 이 모드는 `lane1`, `lane2` 마스크를 따로 보고, 각 마스크의 row별 좌/우 경계 중점을 이어 중심 곡선을 만든 뒤 차량 중심에 더 가까운 곡선을 주행 목표로 씁니다.
- `car`, `traffic_light`는 검출 class에는 포함되지만 조향 geometry에는 쓰지 않습니다.
- 예전 `final.pt`처럼 주행 가능 영역 하나만 뽑는 모델을 다시 쓸 때는 `YOLO_SEGMENTATION_MODE = "drivable_area"`로 바꾸고, `YOLO_CLASS_NAMES`를 모델 class 이름에 맞춰 `("drivable_area",)` 또는 실제 이름으로 바꿉니다.
- area mask 가장자리에 노이즈가 많으면 `YOLO_DRIVABLE_EDGE_PERCENTILE`을 조금 올립니다.
- area 폭이 너무 좁은 row까지 중심선에 쓰이면 `YOLO_DRIVABLE_MIN_ROW_WIDTH_RATIO`를 올립니다.
- 점선을 놓치면 `YOLO_CONFIDENCE`를 낮추고 `YOLO_MIN_VALID_Y_SPAN_RATIO`를 낮춥니다.
- 오검출이 많으면 `YOLO_CONFIDENCE`와 `YOLO_MIN_POINTS_PER_LANE`을 올립니다.
- 점선이 조각조각 잡히면 `YOLO_SKELETON_BRIDGE_GAP_PX`와 `YOLO_DASHED_MERGE_MAX_X_GAP_RATIO`를 조금 올립니다.
- 서로 다른 차선이 한 줄로 합쳐지면 `YOLO_DASHED_MERGE_MAX_X_GAP_RATIO`를 낮춥니다.
- 커브에서 차선 중심이 늦게 따라오면 `YOLO_CURVE_FIT_DEGREE = 2`를 유지합니다. 직선 트랙에서 흔들림이 크면 `1`로 낮춰도 됩니다.

## 제어 튜닝

```python
CONTROL_MODE = "contest"
MIN_SPEED = 200
MAX_SPEED = 200
CONTEST_ANGLE_WEIGHT = 0.7
CONTEST_POSITION_WEIGHT = 0.05
CONTEST_STEERING_ANGLE_NORM_DEG = 50.0
CONTEST_STEER_LIMIT = 10.0
MAX_STEER = 100
REVERSE_STEER = False
```

- 기본 제어는 contest 코드의 `mapped + adjust` 방식입니다.
- 내부 계산은 `mapped = (heading_deg / 50) * 10 * angle_weight`, `adjust = -vehicle_position_x * position_weight`입니다.
- contest의 `vehicle_position_x`는 현재 코드의 `offset_px`와 부호가 반대라서 내부에서 `vehicle_position_x = -offset_px`로 변환합니다.
- 현재 Arduino 펌웨어는 `-100~100`을 좌/우 끝 조향으로 쓰므로, contest 내부 조향값 `-10~10`을 `MAX_STEER` 범위로 다시 스케일링합니다.
- 차가 차선 중심으로 너무 세게 붙으면 `CONTEST_POSITION_WEIGHT`를 낮추고, 커브 방향 반응이 약하면 `CONTEST_ANGLE_WEIGHT`를 올립니다.
- 좌우 조향이 반대로 보이면 `REVERSE_STEER = True`로 바꿉니다.
- 예전 Stanley 제어를 비교하려면 `CONTROL_MODE = "stanley"`로 바꿉니다.

## LiDAR 튜닝

```python
LIDAR_STOP_DISTANCE_MM = 450.0
LIDAR_SLOW_DISTANCE_MM = 900.0
LIDAR_MIN_SPEED_SCALE = 0.35
```

- `STOP_DISTANCE` 안에 장애물이 들어오면 속도는 0이 됩니다.
- `SLOW_DISTANCE` 안에서는 선형으로 감속합니다.
- 차가 너무 늦게 멈추면 두 거리 값을 모두 키웁니다.

## 로그 확인

로그를 켜려면 다음 값을 바꿉니다.

```python
LOG_ENABLED = True
```

또는 실행 중 `l` 키로 토글할 수 있습니다. 로그는 `lane_logs/drive_log.csv`와 `lane_logs/frames/`에 저장됩니다.

차선이 제대로 따지지 않을 때는 `--log`를 켜고 저장된 이미지를 봅니다.

```powershell
python main.py --mode vision --backend yolo --yolo-model models\yolo\final.pt --log --no-arduino
```

- `lane_logs/annotated/`: 화면에 보이는 후보 번호, 선택된 차선 pair, 조향 중심선이 저장됩니다.
- `lane_logs/masks/`: YOLO segmentation mask만 저장됩니다.
- `drive_log.csv`의 `lane_pair` 값이 `pair=right ...`인지 확인합니다. 오른쪽 차로가 아니라면 `YOLO_TARGET_LANE_PAIR`를 `left`, `closest`, `split` 중 하나로 바꿔 비교합니다.
