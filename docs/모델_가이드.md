# 모델 가이드

기존 작업은 Tusimple ResNet18 UFLDv2를 기본으로 사용했습니다. 이번 repo에서는 기본 모델을 CULane ResNet34로 바꿨습니다.

## 변경 이유

- ResNet34는 ResNet18보다 깊은 backbone입니다.
- UFLDv2 공식 README 기준 CULane ResNet18 F1은 75.0, CULane ResNet34 F1은 76.0입니다.
- Tusimple은 비교적 단순한 고속도로 차선 데이터 성격이 강하므로, 실차/경기장 환경에서는 CULane 계열이 더 나은 출발점이 될 수 있습니다.
- 현재 환경은 CUDA를 사용할 수 없으므로 `main.py` 기본값은 CPU입니다.

## 기본 설정

`main.py` 상단 기본값입니다.

```python
LANE_BACKEND = "ufldv2"
UFLDV2_CONFIG_PATH = UFLDV2_REPO_DIR / "configs" / "culane_res34.py"
UFLDV2_MODEL_PATH = BASE_DIR / "models" / "ufldv2" / "culane_res34.pth"
UFLDV2_DEVICE = "cpu"
```

## 모델 다운로드

```powershell
python scripts\download_ufldv2_weights.py --model culane_res34
```

다른 모델을 시험하려면 다음 중 하나를 받을 수 있습니다.

```powershell
python scripts\download_ufldv2_weights.py --model tusimple_res34
python scripts\download_ufldv2_weights.py --model curvelanes_res34
```

그 후 `main.py`의 config와 weight 경로를 맞춰 바꿉니다.

```python
UFLDV2_CONFIG_PATH = UFLDV2_REPO_DIR / "configs" / "curvelanes_res34.py"
UFLDV2_MODEL_PATH = BASE_DIR / "models" / "ufldv2" / "curvelanes_res34.pth"
```

## CPU 환경 주의점

CULane ResNet34는 입력 폭이 1600이라 CPU에서 느릴 수 있습니다. 먼저 다음 순서로 확인하세요.

1. `--backend opencv`로 카메라와 Arduino 방향을 검증합니다.
2. `--mode vision --backend ufldv2`로 딥러닝 overlay가 안정적인지 봅니다.
3. FPS가 낮으면 `UFLDV2_TORCH_THREADS`를 2, 4, 6 중 하나로 바꿔 비교합니다.
4. 그래도 느리면 `culane_res18` 또는 `opencv` backend를 임시 주행용으로 사용합니다.

## OpenCV fallback

`opencv` backend는 딥러닝 모델이 없어도 동작합니다. 실차에서 처음 모터 방향, 카메라 위치, ROI를 맞출 때 사용합니다.

```powershell
python main.py --mode vision --backend opencv
```
