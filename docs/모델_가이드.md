# 모델 가이드

`main` 브랜치의 기본 차선 모델은 YOLO입니다. UFLDv2 구현은 `ufldv2-backup` 브랜치에 보존했습니다.

## 중요한 전제

일반 COCO YOLO 모델은 사람, 자동차, 신호등 같은 객체를 학습했지만 차선 표시는 학습하지 않았습니다. 따라서 `yolo11n.pt`, `yolov8n.pt` 같은 기본 모델을 그대로 쓰면 차선을 안정적으로 잡지 못합니다.

이 repo의 YOLO backend는 아래 둘 중 하나를 기대합니다.

- 차선 segmentation 모델: mask에서 차선 중심선을 추출합니다.
- 차선 detection 모델: box 중심점으로 차선 후보를 추정합니다.

가능하면 segmentation 모델을 쓰는 편이 낫습니다.

YOLO 입력에는 ROI crop을 먼저 적용할 수 있고, 필요하면 Bird-eye view 원근 변환도 적용할 수 있습니다. 이 전처리는 모델 추론 전에만 쓰이며, 조향 계산은 다시 원본 화면 좌표로 복원해서 수행합니다.

Segmentation mask는 바로 선으로 쓰지 않고 아래 순서로 후처리합니다.

1. mask에서 skeleton 중심선을 추출합니다.
2. 점선처럼 끊긴 fragment를 위치 연속성 기준으로 같은 차선 그룹에 묶습니다.
3. 그룹별로 1차 또는 2차 곡선을 fitting합니다.
4. fitting된 곡선을 일정 간격으로 다시 샘플링해서 실선 경로로 사용합니다.

## 모델 파일 위치

기본 경로는 다음과 같습니다.

```text
models/yolo/lane.pt
```

다른 위치의 모델은 실행 시 지정합니다.

```powershell
python main.py --mode vision --backend yolo --yolo-model C:\path\to\lane.pt
```

Ultralytics가 지원하는 `.pt`, `.onnx`, OpenVINO export 디렉터리를 모두 지정할 수 있습니다.

```powershell
python main.py --mode vision --backend yolo --yolo-model models\yolo\yolov8n-seg.onnx
python main.py --mode vision --backend yolo --yolo-model models\yolo\yolov8n-seg_openvino_model
```

## YOLOv8n-seg 범용 포맷

`ctrl_zero` conda 환경에는 YOLO export에 필요한 패키지를 설치했습니다. 현재 로컬 `models/yolo/`에는 아래 산출물이 있습니다.

```text
models/yolo/yolov8n-seg.pt
models/yolo/yolov8n-seg.onnx
models/yolo/yolov8n-seg_openvino_model/
```

다시 생성하려면 다음 명령을 씁니다.

```powershell
conda activate ctrl_zero
python -m pip install -r requirements-export.txt
python scripts\export_yolo_formats.py --model yolov8n-seg.pt
```

주의할 점은 `yolov8n-seg.pt`가 COCO segmentation 기본 모델이라는 것입니다. 포맷 검증과 OpenVINO/ONNX 실행 경로 확인에는 쓸 수 있지만, 차선 표시를 학습한 모델은 아닙니다. 실제 주행에는 차선 데이터로 학습한 segmentation 모델을 같은 방식으로 export해서 쓰는 편이 맞습니다.

## 주요 파라미터

`main.py` 상단에서 조정합니다.

```python
YOLO_MODEL_PATH = BASE_DIR / "models" / "yolo" / "lane.pt"
YOLO_IMAGE_SIZE = 640
YOLO_CONFIDENCE = 0.25
YOLO_IOU = 0.45
YOLO_CLASS_NAMES = ("lane", "left_lane", "right_lane", "center_lane", "dashed_lane", "solid_lane", "road_line", "line")
YOLO_FRAME_SKIP = 1
YOLO_MIN_POINTS_PER_LANE = 3
YOLO_MIN_VALID_Y_SPAN_RATIO = 0.08
YOLO_MASK_SAMPLE_STEP_PX = 6
YOLO_SKELETON_ENABLED = True
YOLO_SKELETON_BRIDGE_GAP_PX = 17
YOLO_DASHED_MERGE_MAX_X_GAP_RATIO = 0.12
YOLO_CURVE_FIT_DEGREE = 2
YOLO_SOLIDIFY_STEP_PX = 6
YOLO_FIT_OUTLIER_REJECTION_PX = 35.0
```

점선 차선이 끊기면 `YOLO_CONFIDENCE`를 조금 낮추거나 `YOLO_MIN_POINTS_PER_LANE`, `YOLO_MIN_VALID_Y_SPAN_RATIO`를 낮춥니다. 오검출이 많으면 반대로 올립니다.

ROI/Bird-eye 전처리 값은 `docs/튜닝_가이드.md`의 "ROI crop / Bird-eye view 튜닝" 절에서 조정합니다.

## FPS 튜닝

YOLO가 느리면 먼저 image size를 낮춥니다.

```python
YOLO_IMAGE_SIZE = 416
```

그래도 부족하면 프레임 스킵을 씁니다.

```python
YOLO_FRAME_SKIP = 2
```

`YOLO_FRAME_SKIP = 2`는 2프레임마다 1번 모델을 돌리고, 중간 프레임은 이전 차선 추정값을 재사용합니다.
